# 同步日志

本目录用于记录每次同步操作的详细信息，包括遇到的问题和解决方案。

## 日志命名规范

格式：`YYYY-MM-DD.md`（例如：`2026-01-27.md`）

## 日志内容模板

每个同步日志应包含以下内容：

### 1. 同步基本信息
- 日期时间
- 执行人（AI Agent 或人工）
- 脚本版本

### 2. 同步概况
- 主模块同步状态
- 子模块同步状态
- 是否遇到问题

### 3. 遇到的问题（如有）
详细描述：
- 问题描述
- 错误信息
- 根本原因分析
- 解决步骤
- 相关 commit hash

### 4. 最终结果
- 同步是否成功
- 新增/修改的文件
- 后续需要注意的事项
